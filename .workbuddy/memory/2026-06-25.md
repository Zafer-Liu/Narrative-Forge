## 修复剧情树无法拖拽（已完成）
- 症状：剧情树节点拖不动。
- 根因：节点用 HTML5 原生拖放（draggable + dragstart/drop），但 `.tree-canvas` 施加了 CSS `zoom` 且位于可滚动绝对定位视口（`.tree-viewport`）中。Chromium/Electron 下原生拖放在 zoom+scroll 场景里坐标/放置目标错乱 → 拖不动。
- 修复（static/src/tree.js）：节点改用 **Pointer Events** 拖拽。`node.draggable=false` + `node.dataset.sceneId`；新增 `bindTreeNodePointerDrag`(pointerdown 记录起点)、window 级 `onTreePointerMove`/`onTreePointerUp`/pointercancel，用 `document.elementFromPoint` + dataset 做屏幕空间命中检测（天然兼容缩放滚动）；5px 移动阈值区分点击/拖拽；同层校验沿用 depthById；拖后置 `treeJustDragged` 吞掉随附 click。`setupTreeNodeDrag()` 在 openTreeBrowser 中调用（带 `_pointerDragBound` 幂等）。移除对 scene-list 的 bindSceneDrag 依赖，改 import moveSceneRelative/renderSceneList。
- CSS（static/style.css）：`.tree-node` 加 `touch-action:none`；新增 `.tree-node.dragging{z-index:5}`、`body.tree-dragging{user-select:none;cursor:grabbing}`。
- 分镜列表（scene-list.js 的 bindSceneDrag）仍用原生拖放，未改动（无 zoom 问题）。
- 构建 bundle v28；前端测试 40 全绿（新增 tests/tree-drag.test.js 5 个：dataset 读取、移动阈值、同层校验）。真实服务器验证 bundle 含 pointermove/tree-dragging。

## 剧情树画板平移失效修复（已完成）
- 症状：剧情树/画板拖不动（平移失效）。与之前节点拖拽是同一类根因。
- 根因：setupTreePan 用 `viewport.scrollLeft/scrollTop` 平移，配合画布 CSS `zoom` 缩放。Electron/Chromium 下 zoom+scroll 叠加导致滚动/坐标错乱 → 拖不动。
- 修复（static/src/tree.js）：画布平移+缩放统一改为纯 CSS `transform: translate(panX,panY) scale(zoom)`。新增 `applyTreeTransform()`；treePanState 改用 panX/panY/originPanX/originPanY（state.js 同步改）；mousedown/touch 记录 origin，move 累加位移后 applyTreeTransform；setTreeZoom 与 renderTreeBrowser 改调 applyTreeTransform（不再写 style.zoom）；openTreeBrowser 重置 panX/panY=0。
- CSS（static/style.css）：`.tree-viewport` overflow auto→hidden；`.tree-canvas` 改 position:absolute + will-change:transform。
- 节点拖拽用 elementFromPoint 屏幕坐标命中，transform 下仍正确，未受影响。
- 构建 bundle v30；前端 40 测试全绿；真实服务器验证 bundle 含 originPanX、已无 scrollLeft、CSS overflow:hidden。
- 教训：本项目缩放画布凡涉及交互（拖节点/平移），一律用 CSS transform，禁用 CSS zoom + scrollLeft 组合。

- app.py：JOBS=JobManager()。export_player/export_serial 改为 JOBS.submit 返回 202+{jobId}；worker 内算 downloadUrl。新增 GET /api/jobs/{id}(api_job, prefix 路由)、POST /api/cancel-job(cancel_job)。serial 的 _ExportCancelled→JobCancelled 别名 import 为 serial_cancelled_error。
