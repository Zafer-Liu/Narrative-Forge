## 添加 Agent+Tool 剧情对话助手

参考 mewcode-python 框架，在项目中实现了完整的 Agent+Tool 框架，风格对齐但无外部依赖。

### 新增文件
- `backend/agent/__init__.py` — 包入口
- `backend/agent/base.py` — Tool 基类、ParamsBase（无 pydantic，用标准库）、ToolRegistry、流事件 dataclass
- `backend/agent/conversation.py` — ConversationManager 多轮对话历史管理
- `backend/agent/client.py` — OpenAICompatClient，requests + SSE 流式，适配 threading HTTP 服务
- `backend/agent/agent.py` — Agent 主循环（同步），yield AgentEvent dict
- `backend/agent/narrative_tools.py` — 剧情助手 4 个工具：get_project_summary、get_scene_detail、update_scene、suggest_branch

### 修改文件

- `static/style.css` — 追加 Agent 面板完整样式
- 执行 npm run build，bundle.js 120.3kb 构建成功

## 供应商配置本地持久化

目标：BaseURL / ModelID / ApiKey 独立存储，不随项目 JSON 走，支持 Railway 环境变量。

### 后端
- `PROVIDER_CONFIG_FILE = PROJECTS_DIR/_provider_config.json`
- `_ENV_TO_FIELD`：14 个环境变量映射到配置字段
- `GET /api/provider-config`：读文件 + 合并 env 覆盖，API Key 用 `__env__` sentinel 不回传
- `POST /api/provider-config`：写文件，env 保护字段不可覆盖

### 前端
- 新增 `static/src/provider-config.js`：`loadProviderConfig()` / `saveProviderConfig()` / `providerSettings(kind)`

- `project-model.js`：`readMetaFromForm` 移除供应商字段；`applyMetaToForm` 移除供应商表单填充；`providerSettings` 代理到新模块；`normalizeProject` 清理旧 meta 里的供应商字段残留
